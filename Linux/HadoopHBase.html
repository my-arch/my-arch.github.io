Configuration for Hadoop and HBase - Oct 2017

Hadoop

  • Dependencies: java 8, openssh, maven (not sure)
    1. Download tar.gz file
    2. Unzip these
    3. Set PATH variables in ~/.bashrc or ~/.profile
    4. Make sure java is java-8
    5. Set JAVA_HOME in etc/hadoop/hadoop-env.sh
    6. Reload bashrc or profile
    7. Configure core-site.xml, hdfs-site.xml, yarn-site.xml, mapred-site.xml
      • core-site.xml - add between <configuration> <property> <name>fs.default.name</name> <value>hdfs://localhost:9000</value> </property>
      • hdfs-site.xml - add between <configuration> <property> <name>dfs.replication</name> <value>1</value> </property> <property> <name>dfs.name.dir</name> <value>file://~/Hadoop/hadoop/hadoop/infra/hdfs/namenode</value> </property> <property> <name>dfs.data.dir</name> <value>file://~/Hadoop/hadoop/hadoop/infra/hdfs/datanode</value> </property>
      • yarn-site.xml - add between <configuration> <property> <name>yarn.nodemanager.aux-services</name> <value>mapreduce_shuffle</value> </property>
      • mapred-site.xml - cp mapred-site.xml.template mapred-site.xml -> add between <configuration> <property> <name>mapreduce.framework.name</name> <value>yarn</value> <property>
    8. Hadoop namenode -format
    9. To start -> run: reload profile, start-dfs.sh and start-yarn.sh

HBase

  1. Download Hbase, tar and make directory 'zookeeper' in hbase folder
  2. Configure hbase-site.xml
    • add between <configuration> <property> <name>hbase.rootdir</name> <value>file://~/Hadoop/hbase-2.0.0-alpha3</value> </property> <property> <name>hbase.zookeeper.property.dataDir</name> <value>~/Hadoop/hbase-2.0.0-alpha3/zookeeper</value> </property>
